# Practise Git

## Introduction

The practice to understand the basis of Git and GitHub. The main purpose of project is to practice the prompts used while working in Git and also understanding the GitHub for collaboration with other people and team.

---

## Project Purpose

This project is created to practice the fundamental workflow of Git version control and to understand how repositories are managed and synchronized using GitHub. The goal is to gain hands-on experience with common Git commands such as repository initialization, staging files, committing changes, and pushing updates to a remote repository.

## Key Step Marks

### Day 8th March 2026 & Day 15th March 2026

First of all after installation of Git software from Git official - created a folder in the drive named "Practise Git", created a file named "Practise-git.html", opened the git bash terminal in the folder named "Practise Git".

The command {git config --global user.name "HaseebK2025"} enter, was used to configure the global Git username. This username is associated with commits and identifies the author of changes in the repository.

The command {git config --global user.email "h***@**.com"} enter, was used to configure the email address connected with the Git account. This allows Git hosting platforms to associate commits with the correct account.

The command {git config --list} enter, was used to verify the configured settings and confirm that the username and email were successfully added.

After exiting the terminal, the git bash terminal was opened again in the folder named "Practise Git"(path still unchanged). The command {git init} enter, was used to initialize a Git repository in the folder so that Git can start tracking changes.

The command {git add .} enter, was used to stage all files in the current directory for the next commit.

The command {git commit -m "Practise Git-8Feb"} enter, was used to create the first commit. A commit records a snapshot of the project files along with a message describing the change.

The command {git branch -M main} enter, was used to rename the current branch to **main**, which is the commonly used default branch name in many repositories.

The command {git remote add orgin https://github.com/HaseebK2025/Practise.git} enter, was used to link the local repository with a remote repository on GitHub so that the project could be uploaded and synchronized online.

The command {git push -u origin main} enter was executed to push the committed files to the remote repository. During this step the authentication window appeared and signed in with browser to grant permission for the push operation.

After exiting the terminal, the file named "Practise-git.html" was updated. The git bash terminal was opened again in the folder named "Practise Git"(path still unchanged).

The command {git init} enter was used again while continuing the practice session.

The command {git add .} enter was used to stage the modified files again.

The command {git commit -m "Repractise 15March"} enter was used to record the updated changes in the repository history with a new commit message.

The command {git push -u origin main}, enter, was used again to upload the new commit to the remote repository.

After exiting the terminal, a text file named "README.md" was created. This file is used to document the project and explain the purpose, workflow, and learning steps of the repository.

The git bash terminal was opened again in the folder named "Practise Git"(path still unchanged). The command {git init}, enter was used again.

The command {git add README.md} enter was used to stage the README documentation file.

The command {git add .} enter was used to ensure that all files were staged for committing.

The command {git commit -m "Practise Guide"} enter was used to save the README documentation and staged files into the repository history.

The command {git push -u origin main} enter was executed to upload the documentation and updates to the remote repository.
